# P4 完整复核报告

## 1. 复核依据

- `15_p4_uiux_design_system.md`
- `16_p4_screen_wireframes.md`
- `17_p4_godot_ui_contract.md`
- Godot 4.6 官方文档中关于 InputMap、Control 焦点和多分辨率 stretch 的说明。

同类 survivor-like 游戏的可用性参考结论：

- 战斗 HUD 应少文本、强数值和状态图标。
- 暂停页承担构筑解释，不把反应说明常驻在战斗 HUD。
- 奖励选择应避免鼠标误点直接领取，特别是风险奖励。
- 手柄/键盘焦点必须始终可见。

## 2. 发现并修复的问题

### 2.1 暂停输入双重处理

问题：`UIRoot` 和 `Main._unhandled_input()` 同时处理 `Esc/P`，恢复暂停时可能被旧逻辑再次切回暂停。

修复：

- 新增 `pause_requested` 信号。
- `UIRoot` 不再直接修改 `game.paused`。
- `Main._unhandled_input()` 只保留 F8/F9/F10 开发快捷键。

### 2.2 奖励鼠标误领取

问题：奖励卡 `pressed` 直接调用领取，违反“鼠标单击聚焦，再通过确认按钮确认”的规则。

修复：

- 奖励卡改为选中态。
- 鼠标点击只聚焦和选中。
- 新增“确认选择”按钮。
- 键盘数字快捷键仍可快速选择，作为可选测试快捷键。

### 2.3 教程流程不完整

问题：手动打开教程后缺少跳过入口，关闭后焦点返回不严格。

修复：

- 三步教程增加“跳过教学”。
- 手动教程记录来源页面，关闭后返回主菜单或暂停页。
- 模态关闭后优先恢复触发控件焦点。

### 2.4 设置项不完整

问题：设置页此前只包含 UI 缩放和低动态模式，不满足显示、音频、操作、可访问性分类。

修复：

- 增加窗口/全屏切换。
- 增加总音量档位。
- 增加键鼠/手柄操作说明。
- 增加屏幕震动、命中停顿、闪屏强度独立设置。
- 低动态模式仍保留一键总开关。

### 2.5 字体未打包

问题：此前依赖 Godot fallback font，不满足“Godot 必须随项目打包中文字体”。

修复：

- 已导入 `assets/fonts/NotoSansSC-VF.ttf`。
- `Main` 绘制文本与 UI Theme 均使用该字体。

### 2.6 Boss HUD 百分比不稳定

问题：Boss 百分比分母使用当前护盾值，会随着护盾减少而变化。

修复：

- 改为使用 `shield_max` + `max_hp`。

### 2.7 旧程序化 UI 残留

问题：旧 `_draw_hud()`、菜单、奖励、修正、教程、暂停和结算绘制函数仍留在 `main.gd`。

修复：

- 已删除旧页面绘制函数。
- 调试信息迁入 `DebugLayer`。

### 2.8 布局锚点过度依赖 1280×720 固定位置

问题：面板位置用绝对坐标，不利于 16:10、21:9 和窗口缩放。

修复：

- 中央面板改为中心锚点。
- HUD 角落面板改为对应边角锚点。
- 设置页改为滚动容器，避免内容溢出。

## 3. 当前仍保留的边界

- 当前 UI 仍集中在 `ui_root.gd` 中程序化创建，不是最终拆分到 `components/`、`hud/`、`screens/` 的生产组件库。
- UI 已使用统一 token 和 Control 节点，但尚未建立独立 `game_theme.tres`。
- 图标是用户批准的暂定版，未来可直接替换同名 PNG。
- 21:9 下世界视野扩展尚未完整实现；当前优先保证 UI 不拉伸和核心信息可读。
- 音频设置已有 UI 和总线控制，但项目尚无正式音效资源。

- Godot 核心模块测试：通过。
- Godot 原运行时冒烟测试：通过。
- Godot UI 运行测试：通过。

## 5. 结论

P4 当前实现已经从“功能型程序化 UI”推进到“符合主要 P4 逻辑的正式 Control UI”。关键误操作风险、输入冲突、字体缺失、设置缺口和旧 UI 残留已经修复。

后续若继续提高到生产级 UI，优先级应为：

1. 将 `ui_root.gd` 拆分为独立组件和页面场景。
2. 建立 `game_theme.tres` 并减少脚本内样式生成。
3. 完整实现 presenter/view-model 边界，进一步减少 UI 对 `Main` 内部数组的直接读取。
4. 配合实际窗口试玩微调布局、字号、奖励卡信息密度和 21:9 世界扩展。
