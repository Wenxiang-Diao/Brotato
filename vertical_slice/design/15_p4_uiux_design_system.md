# P4 阶段 2：UI/UX 统一规范

## 1. 目标

本规范服务于《星链回响》垂直切片和后续完整版本，解决以下问题：

- 战斗时减少阅读负担。
- 奖励选择时突出构筑收益、品质和风险代价。
- 键盘、鼠标、手柄使用同一套焦点逻辑。
- 让 UI 可以由 Figma 规范稳定映射到 Godot `Control`。
- 保留 2D 星尘、晶体和元素反应主题，不复制同类游戏的具体视觉。

## 2. 核心原则

1. **战斗时看状态，不读说明。**  
   HUD 只保留生命、经验、层数、倒计时、技能、武器和当前 Debuff。状态反应说明移入教程与暂停帮助。

2. **选择时先看结论，再看细节。**  
   奖励卡顺序固定为：品质/类型、名称、图标、关键变化、构筑标签、风险代价、输入提示。

3. **颜色不是唯一信息载体。**  
   品质、风险和状态同时使用颜色、图标轮廓、文字标签与边框样式。

4. **危险操作不能与普通确认相同。**  
   返回菜单、重新开始、放弃奖励必须二次确认。

5. **所有页面都必须可用方向键完整导航。**  
   鼠标悬停和手柄焦点使用同一高亮状态，数字键只作为可选快捷键。

## 3. 视觉语言

关键词：

- 深空
- 星图
- 晶体切面
- 清晰轮廓
- 高对比能量色
- 轻量发光，不使用大面积模糊霓虹

形状规则：

- 主面板：低饱和深蓝黑，1 px 冷色边框。
- 卡片：上窄下宽的轻微晶体切角，Godot 首版可先用 8 px 圆角矩形。
- 聚焦态：2 px 高亮边框 + 外侧 2 px 间隔环，不能只改变背景颜色。
- 风险卡：左侧危险色竖条或断裂边框，不把整张卡涂成红色。
- 图标：正视或轻微 3/4 视角，统一粗轮廓，透明背景。

## 4. 色彩 Token

### 4.1 基础与文字

| Token | 色值 | 用途 |
| --- | --- | --- |
| `color/bg/base` | `#080B12` | 游戏 UI 底色 |
| `color/bg/grid` | `#101827` | 星图网格或弱分隔 |
| `color/surface/default` | `#0F1624` | HUD 与普通面板 |
| `color/surface/elevated` | `#17243A` | 卡片、模态窗口 |
| `color/surface/selected` | `#203653` | 聚焦或选中底色 |
| `color/border/default` | `#2A3B56` | 默认边框 |
| `color/border/strong` | `#49698F` | 强调边框 |
| `color/text/primary` | `#F4F7FB` | 标题与正文 |
| `color/text/secondary` | `#B8C5D6` | 次要说明 |
| `color/text/muted` | `#708099` | 低优先信息 |
| `color/overlay/scrim` | `#05070BCC` | 模态遮罩 |

正文颜色在基础底色上的对比度：

- Primary：18.32:1
- Secondary：11.24:1
- Muted：4.91:1

因此 `text/muted` 只允许用于 16 px 以上的非关键文本。

## 5. 字体与排版

推荐字体：

- 中文：`Noto Sans SC` 或 `Source Han Sans SC`。
- 数字与拉丁字符：与中文使用同一家族，避免混排跳动。
- Godot 必须随项目打包字体，不依赖系统字体。

| 样式 | 字号/行高 | 字重 | 用途 |
| --- | --- | --- | --- |
| Display | 52/60 | Bold | 主标题 |
| H1 | 40/48 | Bold | 全屏页面标题 |
| H2 | 32/40 | Bold | 模态标题 |
| H3 | 24/32 | SemiBold | 卡片名称 |
| Body L | 20/30 | Regular | 教程、重要说明 |
| Body M | 18/27 | Regular | 卡片正文 |
| Body S | 16/24 | Regular | HUD、标签、辅助信息 |
| Caption | 14/20 | Medium | 仅用于非关键短标签 |

约束：

- 1280×720 下关键正文不得小于 16 px。
- 单行按钮文字不得缩放，空间不足时按钮加宽或换短文案。
- 数值使用等宽数字特性；计时固定为 `MM:SS`。
- 中文正文每行建议 18 至 28 个汉字。

## 6. 间距、尺寸与层级

间距序列：`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64`。

圆角：

- 小标签：4 px
- 按钮、武器槽：6 px
- 卡片、面板：8 px
- 大模态：12 px

基准尺寸：

- 普通按钮：最小 176×48
- 主按钮：最小 224×56
- 图标按钮：最小 48×48
- 手柄/触控安全目标：不小于 44×44
- 状态图标：32×32
- 武器槽图标：56×56，源文件 128×128
- 奖励卡图标：112×112，源文件 128×128

层级：

1. 战斗世界
2. HUD
3. 战斗提示与飘字
4. 模态遮罩
5. 模态内容
6. Toast 与系统错误
7. 开发调试层

## 7. 组件状态

所有可交互组件必须支持：

- Default
- Hover
- Focused
- Pressed
- Selected
- Disabled
- Dangerous

聚焦态要求：

- 2 px `action/primary` 边框。
- 外侧保留 2 px 间隔，避免与品质边框混淆。
- 至少一个位置、形状或亮度变化。
- 120 至 180 ms 过渡；低动态模式改为瞬时切换。

## 8. 动效与反馈

| 类型 | 时长 | 规则 |
| --- | --- | --- |
| Hover/Focus | 120 ms | 轻微亮度与边框变化 |
| 页面进入 | 180 ms | 透明度 + 8 px 位移 |
| 卡片选中 | 180 ms | 轮廓增强，不进行大幅缩放 |
| 品质揭示 | 240 ms | 金色可延长至 360 ms |
| 错误抖动 | 160 ms | 位移不超过 6 px |

低动态模式：

- 移除界面位移、缩放、持续闪烁和循环发光。
- 保留瞬时颜色、边框和声音反馈。
- 金色品质动画替换为静态金色双边框与“核心机制”标签。

## 9. 声音与输入提示

- 焦点移动、确认、取消、错误、重抽、接受风险使用不同短音效。
- 输入提示根据最近一次有效设备自动切换。
- 键鼠显示 `Enter`、`Esc`、`R` 等键帽。
- 手柄显示确认、取消、肩键等抽象动作，不在逻辑层写死 Xbox/PlayStation 图标。
- 数字键 `1/2/3` 保留为奖励卡快捷键，但不显示为唯一操作方式。

## 10. 可访问性

- 关键正文目标对比度不低于 4.5:1。
- 大字与装饰性文字目标不低于 3:1。
- 品质与风险必须有文字标签或图形差异。
- 所有功能可仅用键盘完成。
- 焦点不得被遮罩后的控件获取。
- 模态打开时焦点进入首个安全选项；关闭后返回触发控件。
- 支持低动态、屏幕震动、命中停顿和闪屏强度设置。
- 支持 UI 缩放：90%、100%、110%、125%。

## 11. 文案规则

- 按钮使用动作：`开始挑战`、`继续`、`重新开始`。
- 风险说明使用“收益 / 代价 / 可修正 / 构筑适配”四段固定结构。
- 不使用仅开发者理解的 `Debuff` 作为主标签；界面显示“负面状态”，详情可保留术语。
- 错误文案说明后果和恢复动作，不显示内部路径。
- 构建版本放在菜单角落，不再显示“P1 试玩就绪版”。
