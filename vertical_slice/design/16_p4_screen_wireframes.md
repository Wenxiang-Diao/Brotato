# P4 阶段 2：页面结构与线框

## 1. 基准画布

- 设计分辨率：1280×720。
- 基础安全区：四边 32 px。
- HUD 紧凑安全区：四边 24 px。
- 全屏内容最大宽度：1120 px。
- 模态内容最大宽度：960 px。
- 支持比例：16:9、16:10、21:9。
- 超宽屏只扩展游戏视野，主要 UI 保持在中央 16:9 安全区。

## 2. 主菜单

```text
----------------------------------------------------------------+
| build v0.x                                            [设置]   |
|                                                                |
|                       星 链 回 响                              |
|                   元素构筑生存动作游戏                          |
|                                                                |
|              [ 开始标准挑战 ]                                  |
|              [ 开始风险挑战 ]  风险标签                         |
|              [ 战斗说明 ]                                      |
|                                                                |
|             最近输入设备提示 / 确认与返回                        |
+----------------------------------------------------------------+
```

规则：

- 默认焦点为标准模式。
- 风险模式旁显示一句短说明，不在主菜单展开完整 Debuff 规则。
- 设置可由鼠标点击或肩键/Tab 快速到达。
- 固定种子只在开发构建或高级设置中显示。

## 3. 首次战斗引导

引导改为三步：

1. 移动与走位。
2. 武器自动攻击，主动技能需要手动释放。
3. 标记、感电、冻结、灼烧可以组成三种反应。

```text
+--------------------------------------------------------------+
|                      战斗说明  1 / 3                          |
|                                                              |
|                    [输入设备示意图]                           |
|                                                              |
|                 使用方向输入移动与躲避                        |
|                                                              |
|       [跳过教学]                        [下一步]               |
+--------------------------------------------------------------+
```

规则：

- 每页只说明一个概念。
- 输入设备切换时提示图自动更新。
- 可跳过，并在暂停菜单“战斗说明”中重新查看。

## 4. 战斗 HUD

```text
+----------------------------------------------------------------+
| [HP 数值================]      第 1/6 层      [00:42]            |
| [XP ========           ]      标准模式                            |
| Lv.3                                                           |
|                                                                |
|                                                                |
|                         战斗区域                                |
|                                                                |
|                                                                |
| [武器][武器][武器][武器]           [负面状态图标 x 层数]         |
| [主动技能图标  冷却环]                 [暂停提示]                 |
+----------------------------------------------------------------+
```

规则：

- HP、XP 和层数属于一级信息。
- 武器只显示图标、等级与冷却状态；详细名称在暂停构筑页显示。
- Debuff 使用图标、层数和危险边框；悬停/聚焦后才显示详情。
- 状态反应图例不常驻。
- F8/F10 等开发提示不出现在正式 HUD。
- Boss 出现时，顶部中央切换为 Boss 名称、阶段和护盾/生命条。

## 5. 奖励三选一

```text
+----------------------------------------------------------------+
|                      升级：选择一项                             |
|            当前构筑：[标记] [感电] [冻结]                       |
|                                                                |
| +--------------+ +--------------+ +--------------+              |
| | [品质][类型] | | [品质][类型] | | [风险][类型] |              |
| |    图标      | |    图标      | |    图标      |              |
| | 奖励名称     | | 奖励名称     | | 奖励名称     |              |
| | 关键变化     | | 关键变化     | | 关键变化     |              |
| | 构筑标签     | | 构筑标签     | | 构筑标签     |              |
| |              | |              | | --------------|              |
| |              | |              | | 代价与等级    |              |
| +--------------+ +--------------+ +--------------+              |
| [查看详情]       [重抽  2]                    [确认选择]          |
+----------------------------------------------------------------+
```

卡片内容顺序：

1. 品质与奖励类型。
2. 128×128 图标。
3. 奖励名称。
4. 一句关键结果，例如“解锁感电弹射武器”。
5. 当前等级变化或持有状态。
6. 构筑标签，例如“感电 / 连锁”。
7. 风险卡附加危险等级、代价、可修正性和适配提示。

导航：

- 左右切换卡片。
- 确认键选择，取消键返回详情层。
- `R` 或手柄指定动作重抽。
- 鼠标双击不直接确认；单击聚焦，再点击确认按钮。

## 6. 五级修正窗口

```text
+--------------------------------------------------------------+
|                     负面状态修正                              |
|                                                              |
| 当前状态： [图标] 电荷反噬 I   轻度   可移除                  |
| 来源：奖励“雷纹扩散”                                         |
| 影响：未触发雷链时，下一次受伤提高                            |
|                                                              |
| [ 移除该状态 ]                                                |
| [ 保留风险，恢复生命 ]                                        |
| [ 保留风险，获得经验 ]                                        |
|                                                              |
| 构筑提示：当前感电强化较多，保留风险仍可控                    |
+--------------------------------------------------------------+
```

规则：

- 不能只显示“移除最早 Debuff”，必须显示实际将被移除的状态。
- 安全选项默认聚焦“移除”。
- 保留风险选项用 warning，不使用 danger，除非会造成即时失败。

## 7. 暂停与设置

暂停主层：

```text
+----------------------+---------------------------------------+
| 继续                 | 当前构筑                              |
| 战斗说明             | [4 个武器槽：图标、名称、等级]       |
| 设置                 | [状态与反应摘要]                     |
| 重新开始             | [负面状态：危险等级、来源、可修正]   |
| 返回主菜单           |                                       |
+----------------------+---------------------------------------+
```

设置分类：

- 显示：全屏、分辨率、UI 缩放。
- 音频：总音量、音乐、音效。
- 操作：键位与手柄提示。
- 可访问性：低动态、屏幕震动、命中停顿、闪屏强度。

重新开始和返回主菜单必须打开确认弹窗。

## 8. 结算页

```text
+----------------------------------------------------------------+
|                       挑战失败 / 胜利                           |
|              标准模式 | 第 6/6 层 | 等级 14                     |
|                                                                |
| [构筑总结]                    [战斗统计]                         |
| 武器图标 x4                   击杀 / 伤害 / 受伤                 |
| 主要状态与 Debuff             雷链 / 破碎 / 热冲击               |
| 本局亮点                      最长存活 / Boss 阶段                |
|                                                                |
|             [ 再次挑战 ]   [ 返回主菜单 ]                       |
+----------------------------------------------------------------+
```

规则：

- 主要按钮为“再次挑战”。
- 数据保存错误使用非阻断 Toast，不占据结算页主信息位置。
- 胜利与失败共享结构，仅改变标题、色彩和本局亮点。

## 9. 焦点导航

全局规则：

- 页面打开时焦点必须可见。
- 横向卡片使用左右，纵向菜单使用上下。
- 不允许循环焦点突然跨越大区域。
- 弹窗关闭后焦点返回触发按钮。
- 鼠标移动到控件上时更新焦点，但不自动确认。

快捷键：

| Action | 键盘默认 | 手柄抽象 |
| --- | --- | --- |
| `ui_accept` | Enter / Space | Face Bottom |
| `ui_cancel` | Esc | Face Right |
| `ui_reroll` | R | Face Left |
| `ui_details` | Tab / F | Face Top |
| `ui_pause` | Esc / P | Start |
| `active_skill` | Space | Right Trigger |

`Space` 在战斗中是主动技能，在菜单中可作为确认；状态上下文必须互斥。

## 10. 适配策略

- 16:10：增加上下留白，不拉伸控件。
- 21:9：HUD 固定在中央安全区边缘，战斗世界扩展。
- 低于 1280×720：整体缩放至 90%，禁止继续缩小正文。
- 语言扩展：按钮预留中文文案宽度的 1.4 倍。
- 卡片正文超过 4 行时进入详情层，不压缩字体。
