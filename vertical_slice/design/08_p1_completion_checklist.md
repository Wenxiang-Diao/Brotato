# P1 试玩就绪检查表

> 启动日期：2026-06-07

## 目标

P1 不扩充垂直切片的内容数量，重点把 P0 原型提升为可交给外部测试者独立试玩、理解和复盘的版本。

## 实现顺序

### 1. 新手引导与暂停

- [x] 首次进入战斗时显示操作和目标说明。
- [x] 支持暂停、继续和返回模式选择。
- [x] 暂停时停止战斗计时与模拟。

### 2. 战斗预警和状态可读性

- [x] Boss 范围攻击具有倒计时或收缩预警。
- [x] HUD 显示四种状态和三种反应的简短图例。
- [x] 反应触发具有独立颜色和文字反馈。

### 3. 奖励重抽与构筑信息

- [x] 每局提供有限次数的奖励重抽。
- [x] 奖励界面显示类型、风险代价和剩余重抽次数。
- [x] HUD 显示武器等级，而不是只显示武器 ID。

### 4. 结算统计与复玩入口

- [x] 胜利和失败界面显示模式、层数、等级、击杀、伤害和反应次数。
- [x] 支持直接以相同模式重新开始。
- [x] 保留返回模式选择入口。

### 5. 验证与性能

- [x] Godot 解析和运行时冒烟测试通过。
- [x] 自动测试覆盖暂停、重抽、结算与两种模式。
- [x] 数据校验和 Python 测试通过。
- [x] 150 敌人、100 投射物上限不被新功能破坏。

## P1 明确不做

- 程序化路线分支、事件节点和商店。
- 水晶树、局外成长和长期存档。
- 新角色、新武器、新敌人和新 Boss。
- 正式美术、音效和动画资源。
- 武器随机词条、品质系统和合成。

这些内容需要先获得真人 A/B 测试结果，再决定是否进入 P2。
