# P4 精简 HUD 与奖励界面复核报告

## 1. 修改目标

本轮修改用于解决真人测试前的界面阻塞问题：

- 战斗 HUD 占用面积过大。
- 负面状态使用独立大面板，即使为空也长期占据视野。
- 武器、技能和 Debuff 信息分散。
- 奖励图标被拉伸，卡片缺少清晰的简要说明。
- 暂停页没有承担足够的构筑解释。
- 设置页缺少按键重映射。

## 2. 同类游戏参考原则

参考：

- Brotato Steam 页面与官方截图：
  `https://store.steampowered.com/app/1942280/Brotato/`
- Vampire Survivors Steam 页面与官方截图：
  `https://store.steampowered.com/app/1794680/Vampire_Survivors/`
- Halls of Torment Steam 页面与官方截图：
  `https://store.steampowered.com/app/2218750/Halls_of_Torment/`

共同原则：

- 战斗时只保留生命、时间、进度和当前构筑的快速识别信息。
- 长文本、构筑解释和状态说明进入暂停页或选择页。
- 升级选择需要让玩家在不阅读长段落时先理解核心收益。
- 鼠标悬停、焦点移动或详情区域用于承载完整描述。
- 多输入游戏必须允许玩家确认当前操作绑定。

## 3. 已完成修改

### 3.1 战斗 HUD

- 左上角仅保留生命、经验和等级。
- 顶部中央仅保留时间、层数和模式。
- 武器、主动技能和 Debuff 合并为底部单一构筑条。
- 没有 Debuff 时不显示空状态栏。
- 每个构筑项使用固定尺寸图标，上方不再出现大面积说明文字。
- 悬停图标可查看名称、等级或 Debuff 影响。
- 修复空 Toast 面板长期显示的问题。

### 3.2 奖励界面

- 奖励卡改为固定图标区，不再拉伸图片填满整张卡。
- 图标下方直接显示名称、类型和核心收益。
- 风险卡直接显示风险等级。
- 聚焦或悬停卡片时，下方详情框显示完整描述、武器基础信息和 Debuff 代价。
- 领取仍必须通过“确认选择”按钮，卡片点击只负责选择。

### 3.3 暂停界面

- 暂停页承担详细构筑说明。
- 显示武器名称、等级和中文类型。
- 显示每个 Debuff 的名称与完整影响。
- 显示三种状态反应组合。

### 3.4 按键设置

- 设置页新增移动、主动技能和暂停按键重映射。
- 新绑定即时生效并写入 `user://input_bindings.cfg`。
- 重启游戏后自动恢复绑定。
- 新绑定会移除其他动作上的相同按键，避免一键触发多个动作。
- 支持恢复默认按键。

## 5. 当前边界

- HUD 图标仍使用暂定资产。
- 按键重映射当前覆盖键盘；手柄按键重映射尚未进入本轮范围。
- 设置页使用滚动容器，较低分辨率下需要滚动查看全部绑定。
- 暂停页构筑说明仍由一个只读文本摘要生成，后续可升级为独立武器和状态组件。
