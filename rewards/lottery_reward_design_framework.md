# 🎲 抽奖 / 奖励系统框架

> 版本：v1.0  
> 优先级：P0  
> 目标：将局内战斗、节点路线、元素状态、武器构筑、debuff 风险和水晶成长串联为可重复游玩的核心循环。

---

## 1. 系统目标

抽奖 / 奖励系统需要承担四个功能：

1. **提供局内成长**：让玩家通过持续选择形成战斗力提升。
2. **驱动 Build 成型**：让武器、状态、角色技能和水晶方向产生联动。
3. **制造风险取舍**：高价值奖励往往伴随 debuff 或资源成本。
4. **强化节点路线意义**：不同路线对应不同奖励预期，避免路线只是假选择。

---

## 2. 核心循环

```text
战斗 / 节点完成
→ 获得经验、金币、节点奖励
→ 触发抽奖或奖励选择
→ 玩家选择武器 / 状态 / Buff / 修正 / 资源
→ 可能获得 Debuff 或触发风险条件
→ Build 方向发生变化
→ 进入下一节点
```

---

## 3. 抽奖来源

| 来源 | 触发频率 | 奖励价值 | 风险程度 | 说明 |
|---|---:|---:|---:|---|
| 升级抽奖 | 高频 | 中低 | 低 | 最基础的构筑来源 |
| 节点奖励 | 中频 | 中 | 中 | 根据节点类型变化 |
| 精英奖励 | 低频 | 高 | 中高 | 重要构筑转折点 |
| Boss 奖励 | 章节末 | 很高 | 中 | 阶段性成型与解锁 |
| 商店购买 | 可控 | 取决于金币 | 低中 | 玩家主动规划 |
| 事件奖励 | 不稳定 | 波动大 | 高 | 风险收益与特殊构筑 |
| 休息修正 | 低频 | 防御型 | 低 | 修复风险、稳定路线 |

---

## 4. 奖励选择形式

### 4.1 标准三选一

最常见奖励形式。

```text
奖励选项 A：稳健成长
奖励选项 B：Build 强化
奖励选项 C：高风险高收益
```

设计目的：让玩家每次选择都能围绕当前局势进行判断。

---

### 4.2 商店陈列

商店提供 6 个货架位：

| 槽位 | 内容 |
|---|---|
| 1-2 | 武器 / 武器升阶 |
| 3-4 | Buff / 状态核心 |
| 5 | Debuff 修正 / 回复 |
| 6 | 特殊折扣或风险商品 |

---

### 4.3 事件二选一 / 三选一

事件节点更强调不确定性，常见结构：

```text
安全选择：低收益，无风险
构筑选择：中收益，轻微风险
贪婪选择：高收益，附带 debuff 或战斗挑战
```

---

## 5. 奖励标签体系

每个奖励都应拥有标签，便于权重系统识别。

| 标签 | 说明 |
|---|---|
| `weapon` | 武器或武器升阶 |
| `status` | 状态核心或状态强化 |
| `burn` | 灼烧相关 |
| `poison` | 中毒相关 |
| `freeze` | 冻结相关 |
| `shock` | 感电相关 |
| `mark` | 标记相关 |
| `armor_break` | 破甲相关 |
| `control` | 控制流 |
| `chain` | 连锁流 |
| `dot` | 持续伤害流 |
| `burst` | 爆发流 |
| `survival` | 生存 / 回复 |
| `risk` | 风险奖励 |
| `debuff_fix` | debuff 修正 |
| `crystal` | 水晶 / 局外成长 |
| `role_skill` | 角色技能强化 |

---

## 6. 与角色系统的关系

角色不直接决定奖励结果，但会影响奖励权重。

示例：

| 角色倾向 | 权重变化 |
|---|---|
| 新手均衡型 | 基础武器、回复、通用 Buff 权重提高 |
| 状态连锁型 | 感电、标记、扩散、连锁标签权重提高 |
| 风险收益型 | 低血量、过载、debuff 转收益奖励权重提高 |
| 控制生存型 | 冻结、吸附、护盾、回复权重提高 |

---

## 7. 与水晶系统的关系

水晶系统提供局外长期方向，但不能完全决定局内奖励。

设计原则：

- 水晶只调整权重，不直接锁定奖励。
- 水晶加成需要有上限，防止每局 Build 过于固定。
- 高级水晶可以解锁特殊奖励池，但出现仍需概率判断。

---

## 8. 与 Debuff 系统的关系

Debuff 是高收益奖励的成本来源。

常见绑定方式：

| 奖励强度 | Debuff 绑定 |
|---|---|
| 蓝 / 紫 | 通常无 debuff |
| 粉 | 小概率绑定轻度 debuff |
| 红 | 常见绑定中度 debuff 或条件限制 |
| 金 | 必须绑定代价、限制、挑战或稀缺资源 |

---

## 9. MVP 实现边界

MVP 不需要复杂 AI 推荐系统，只需要实现：

1. 奖励池标签
2. 品质概率
3. 角色倾向权重
4. 当前 Build 简单权重
5. 水晶权重加成
6. 重抽与锁定
7. 高品质奖励绑定 debuff

后续版本再加入更复杂的 Build 识别和动态补偿。
