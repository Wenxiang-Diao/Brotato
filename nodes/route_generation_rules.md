# 路线生成规则

---

## 1. 基础生成目标

路线生成需要满足三个目标：

1. 每局都有随机性；
2. 玩家能做出有意义的路线选择；
3. 不出现明显不公平或无解路线。

---

## 2. 路线结构参数

| 参数 | MVP 推荐值 |
|---|---:|
| 总层数 | 18 |
| 章节数 | 3 |
| 每章层数 | 6 |
| 每层节点数 | 2-4 |
| 最大可见未来层数 | 2 |
| Boss 层 | 6、12、18 |
| 每章精英节点数量 | 1-2 |
| 每章商店节点数量 | 1 |
| 每章休息节点数量 | 0-1 |

## 4. 强制节点规则

| 规则 | 说明 |
|---|---|
| 第 1 层固定普通战斗 | 保证玩家熟悉操作 |
| Boss 层固定 | 第 6、12、18 层固定 Boss |
| Boss 前一层至少出现商店、休息或抽奖之一 | 给玩家最终准备机会 |
| 每章至少 1 个事件节点 | 保证随机体验 |
| 每章至少 1 个抽奖节点 | 保证 build 推进 |
| 第 2 章后每章至少 1 个精英节点 | 保证高风险路线存在 |

---

## 5. 重复限制

为了避免路线体验单调：

- 同一路径连续普通战斗节点不超过 3 个；
- 同一路径连续非战斗节点不超过 2 个；
- 精英节点不能连续出现；
- 休息节点不能连续出现；
- 商店节点不能连续出现；
- 风险节点不能出现在第 1 章前 4 层；
- Boss 前一层不能全部是高风险节点。

---

## 6. 分支连接规则

### 6.1 基础连接

每层节点连接到下一层节点：

- 每个节点至少连接 1 个下一层节点；
- 每个节点最多连接 2 个下一层节点；
- 每个下一层节点至少有 1 个来源；
- 避免所有路线在中途强制合流。

---

### 6.2 分支策略

建议每章至少形成 3 类路线：

| 路线 | 特征 |
|---|---|
| 稳定路线 | 普通战斗 + 休息 + 商店 |
| 成长路线 | 抽奖 + 事件 + 普通战斗 |
| 风险路线 | 精英 + 风险 + 宝藏 |

玩家可以根据当前状态选择：

- 血量低：稳定路线；
- build 缺关键组件：成长路线；
- build 已成型：风险路线。

---

## 7. 动态修正规则

根据玩家当前状态，系统可以轻微调整未来节点权重。

| 玩家状态 | 调整方向 |
|---|---|
| 血量低于 30% | 休息节点权重小幅提升 |
| debuff 数量 ≥ 3 | 商店 / 休息权重提升 |
| 长时间未获得武器 | 抽奖节点权重提升 |
| 已连续 2 个非战斗节点 | 普通战斗权重提升 |
| 已挑战精英成功 | 风险节点权重小幅提升 |

注意：动态修正只影响权重，不应完全保底，以免削弱 Roguelite 随机性。

## 8. 伪代码示例

```pseudo
GenerateRoute(seed):
    route = []
    for chapter in 1..3:
        for layer in 1..6:
            globalLayer = (chapter - 1) * 6 + layer

            if globalLayer in [6, 12, 18]:
                nodes = [BossNode(chapter)]
            else if globalLayer == 1:
                nodes = [NormalBattleNode, NormalBattleNode]
            else:
                weights = GetChapterWeights(chapter)
                weights = ApplyForcedRules(weights, globalLayer)
                weights = ApplyRepeatLimits(weights, route)
                weights = ApplyDynamicAdjustment(weights, playerState)
                nodes = SampleNodes(weights, nodeCountByChapter(chapter))

            route.append(nodes)

    ConnectLayers(route)
    ValidateRoute(route)
    return route
```

---

## 9. 路线合法性检查

生成后需要检查：

- 是否存在 18 层；
- Boss 层是否固定；
- 每章是否有抽奖节点；
- 每章是否有事件节点；
- Boss 前是否有准备选项；
- 是否存在死路；
- 是否存在连续过多重复节点；
- 是否存在全路线过难或过软。
