# 🌳 水晶 / 局外成长系统框架

---

## 1. 设计定位

水晶系统是《星链回响》的局外成长核心，对应总策划中的“古树水晶”。

它不应该只是传统意义上的永久加点，而应该承担以下职责：

- 让玩家在每局失败或通关后仍有推进感。
- 让玩家能逐渐塑造自己偏好的构筑方向。
- 让元素系统、抽奖系统、Debuff 系统、节点系统形成闭环。
- 给不同角色提供长期成长差异。

---

## 2. 设计原则

### 2.1 机制优先，数值克制

水晶节点可以提供数值，但不能以纯数值堆叠为主。

推荐：

- 提高某类奖励出现权重
- 解锁特殊修正选项
- 改变某个状态交互的触发方式
- 提高路线选择信息量
- 降低特定风险的失控概率

谨慎：

- 永久 +攻击力
- 永久 +生命值
- 永久 +暴击率
- 永久 +伤害倍率

### 2.2 改变可能性，不直接给答案

水晶可以让“感电 build 更容易成型”，但不能让玩家开局必定获得完整感电 build。

### 2.3 成长有取舍

玩家不能同时轻松点满所有方向。不同分支应对应不同玩法偏好。

### 2.4 支持重置

由于本项目强调构筑探索，水晶系统必须支持低成本或阶段性重置，避免玩家早期误点后体验受损。

---

## 3. 核心资源

| 资源 | 获得方式 | 用途 |
|---|---|---|
| 星尘 | 每局结算、普通节点、成就 | 升级通用节点 |
| 元素晶片 | 触发对应元素反应、击败元素怪物 | 升级元素分支节点 |
| 古树芽核 | Boss、章节通关、首次挑战奖励 | 解锁关键节点 / 终点节点 |
| 羁绊露滴 | 使用角色完成挑战、角色剧情 | 解锁角色联动节点 |
| 裂隙残片 | 高风险节点、诅咒奖励 | 解锁风险相关节点 |

---

## 4. 水晶树结构

整体采用“六分支 + 星核中心”的结构。

```text
             导能
              |
生息 —— 星核 —— 灼焰
              |
             凝流
              |
             结晶
```

实际 UI 可以设计成古树根系或星盘结构：

- 中心为星核水晶
- 五个元素分支向外延展
- 每个分支有 3 层：基础层、构筑层、核心层
- 关键节点需要古树芽核解锁

---

## 5. 节点类型

| 类型 | 作用 | 示例 |
|---|---|---|
| 权重节点 | 改变抽奖池倾向 | 感电奖励出现权重 +5% |
| 机制节点 | 改变状态或构筑机制 | 首次感电连锁额外跳跃 1 次 |
| 修正节点 | 降低风险失控 | 每局首次 Debuff 替换免费 |
| 路线节点 | 改善节点路线信息 | 提前预览下一层一个节点奖励类型 |
| 资源节点 | 改变资源收益 | Boss 额外掉落对应元素晶片 |
| 羁绊节点 | 与角色成长相关 | 指定角色初始获得一次元素倾向修正 |
| 终点节点 | 分支核心能力 | 解锁该分支独特局内事件或奖励 |

---

## 6. 与现有系统的关系

| 系统 | 水晶影响方式 |
|---|---|
| 元素 / 状态 | 提高状态触发稳定性、强化特定交互 |
| 武器 | 提高对应武器标签奖励权重，但不直接给武器 |
| 角色 | 角色羁绊解锁专属水晶修正 |
| 怪物 | 某些怪物掉落对应元素晶片 |
| 节点 | 解锁路线预览、特殊水晶节点、风险节点收益 |
| 抽奖 / 奖励 | 改变奖励权重、保底类型、重抽选项 |
| Debuff | 提供修正、压制、替换、转化能力 |

---

## 7. 成长节奏

### 早期：方向选择

玩家主要解锁星核基础节点和一个元素分支。

### 中期：构筑强化

玩家开始投资关键节点，形成偏好的 build 方向。

### 后期：风险优化

玩家解锁高风险高收益节点，用于挑战更高难度。

---

## 8. 系统边界

水晶系统不应该：

- 直接决定一局的最终 build。
- 大幅提高基础数值导致前期无压力。
- 让玩家跳过 Debuff 风险。
- 让单一元素分支永远优于其他分支。
- 让新玩家一上来面对复杂树状系统而迷失。
