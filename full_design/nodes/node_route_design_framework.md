# 节点 / 关卡路线系统设计框架

---

## 1. 系统目标

节点 / 关卡路线系统的核心目标是：

> 让玩家在每一局中通过路线选择主动塑造自己的成长节奏、风险水平和 build 方向。

本项目的核心不是固定场地无限刷怪，而是通过节点推进形成更强的策略层：

```text
当前状态判断 → 路线选择 → 节点挑战 → 奖励选择 → 风险修正 → 下一节点
```

---

## 2. 设计原则

### 2.1 节点必须服务 build

每个节点都应回答一个问题：

> 这个节点如何帮助玩家形成、修正或强化 build？

节点不能只给金币或经验，而要影响至少一种构筑维度：

- 武器数量 / 品质；
- 元素或状态倾向；
- buff / debuff 组合；
- 水晶成长；
- 角色技能发挥；
- 怪物应对方式。

---

### 2.2 风险和收益必须绑定

高收益节点必须有明确代价：

| 高收益形式 | 对应风险 |
|---|---|
| 高品质武器 | 精英怪 / 强化怪群 |
| 稀有 buff | 获得 debuff |
| 水晶碎片 | 当前局战斗收益较低 |
| 移除 debuff | 放弃一次成长机会 |
| 特殊事件奖励 | 血量、金币、随机负面效果代价 |

---

### 2.3 路线选择要可预判但不完全透明

玩家应能看到下一层或未来 2 层的节点类型，但不应完全知道所有事件结果。

推荐信息透明度：

| 信息 | 是否显示 |
|---|---|
| 节点类型 | 显示 |
| 基础奖励类型 | 显示 |
| 敌人主题 | 可显示 |
| 事件具体结果 | 不完全显示 |
| 精英词缀 | 可在进入前显示 |
| Boss 名称 | 章节末显示 |

---

### 2.4 短局节奏优先

MVP 一局推荐 18 层，单局时长控制在 20-35 分钟。

| 层段 | 目标体验 |
|---|---|
| 1-3 | 快速建立基础武器和状态方向 |
| 4-6 | 第一次 build 检验，章节 Boss |
| 7-10 | 路线分化，出现高风险奖励 |
| 11-12 | 第二次 build 检验 |
| 13-17 | 高压强化和最终准备 |
| 18 | 最终 Boss |

---

## 3. 基础路线结构

### 3.1 层数结构

```text
第 1 章：1 - 6 层
第 2 章：7 - 12 层
第 3 章：13 - 18 层
```

每章 6 层：

- 前 4 层：路线选择和 build 推进；
- 第 5 层：准备层或高压层；
- 第 6 层：Boss 层。

---

### 3.2 分支结构

每一层推荐生成 2-4 个节点，玩家每次只能选择一个。

推荐结构：

| 阶段 | 每层节点数 | 设计意图 |
|---|---:|---|
| 第 1 章 | 2-3 | 降低理解压力 |
| 第 2 章 | 3-4 | 增加路线策略 |
| 第 3 章 | 3-4 | 强化风险选择 |

---

### 3.3 连接规则

- 每个节点至少连接到下一层 1 个节点；
- 每个节点最多连接到下一层 2 个节点；
- 不允许出现完全无选择的长链超过 2 层；
- 不允许同一路径连续出现 3 个同类非战斗节点；
- Boss 层固定为单节点。

---

## 4. 节点和系统关系

| 系统 | 节点系统中的表现 |
|---|---|
| 战斗系统 | 普通、精英、Boss 节点提供战斗压力 |
| 元素系统 | 奖励倾向、事件、怪物抗性影响元素选择 |
| 武器系统 | 抽奖、商店、精英奖励提供武器获取和升阶 |
| debuff 系统 | 风险节点、事件、强力奖励绑定 debuff |
| 水晶系统 | 宝藏节点、Boss 奖励提供水晶碎片 |
| 角色系统 | 角色倾向影响推荐路线和节点收益 |
| 怪物系统 | 不同节点投放不同怪物组合与精英词缀 |

---

## 5. MVP 版本目标

MVP 不追求复杂地图，而追求清晰可玩：

- 18 层路线；
- 3 个章节；
- 7 类基础节点；
- 3 个 Boss；
- 12 个事件；
- 简单节点权重；
- 简单风险收益系统；
- 可视化路线图。

后续再扩展：

- 多地图主题；
- 角色专属路线事件；
- 元素异常天气；
- 隐藏 Boss；
- Endless 路线；
- DLC 章节。
