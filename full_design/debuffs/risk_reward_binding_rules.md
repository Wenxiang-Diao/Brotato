# 风险奖励绑定规则

---

## 1. 设计目的

风险奖励绑定用于解决 Roguelite 中常见的“只选最强数值奖励”的问题。

本项目的奖励系统强调构筑和状态联动，因此强力奖励应带来明确代价，让玩家在以下问题中做取舍：

```text
我要更快成型，还是保持稳定？
我要强化当前 build，还是避免被 Debuff 限制？
我要接受短期风险，还是等待后续更合适的选项？
```

---

## 2. 奖励品质与 Debuff 绑定

| 奖励品质 | 默认 Debuff 绑定概率 | 说明 |
|---|---:|---|
| 蓝 | 0% | 基础奖励，帮助起步 |
| 紫 | 15% | 小幅强化，可绑定轻度 Debuff |
| 粉 | 35% | 构筑强化，常见轻 / 中度 Debuff |
| 红 | 65% | 强核心奖励，常绑定中 / 重度 Debuff |
| 金 | 85% | 极强奖励，可能绑定诅咒 |

该概率可被水晶、角色、节点类型修正。

---

## 3. 奖励类型与适配 Debuff

| 奖励类型 | 适合绑定的 Debuff |
|---|---|
| 高攻击奖励 | 防御降低、暴击风险、资源下降 |
| 高频攻击奖励 | 移速降低、状态反噬、技能冷却增加 |
| 大范围 AOE | 击退降低、单体伤害下降、精英抗性提高 |
| 状态强化 | 自身易受状态、状态持续不稳定 |
| 连锁强化 | 未触发连锁时伤害下降 |
| 吸血 / 回复 | 最大生命降低、治疗衰减 |
| 高品质武器 | 商店价格提高、经验需求提高 |
| 水晶碎片奖励 | 下一节点敌人强化、当前 Debuff 升层 |

---

## 4. 绑定强度公式建议

MVP 阶段可以用规则表，不必实现复杂公式。

设计上可按以下思路估算：

```text
奖励强度 = 品质系数 + 构筑核心系数 + 泛用性系数
Debuff 强度 = 奖励强度 × 风险倍率
```

| 奖励泛用性 | Debuff 要求 |
|---|---|
| 泛用强 | Debuff 必须更明显 |
| 条件强 | Debuff 可较轻 |
| 单流派强 | Debuff 与该流派形成取舍 |
| 高操作强 | Debuff 可偏操作或资源 |

---

## 5. 三选一中的风险展示

抽奖界面必须清楚展示：

- 奖励效果
- 是否绑定 Debuff
- Debuff 名称
- Debuff 危险等级
- 是否可在修正窗口移除
- 是否与当前角色 / build 有联动提示

推荐显示方式：

```text
红色奖励：灼焰核心
效果：灼烧反应伤害 +40%
代价：获得「易燃体质 II」
提示：你当前已拥有 4 个灼烧相关强化，该奖励将显著提高输出，但会增加受到灼烧伤害。
```

---

## 6. 风险奖励的保底规则

避免玩家被迫接受过高风险：

1. 三选一中至少 1 个选项不绑定 Debuff。
2. 若玩家已有 4 个以上 Debuff，下一次抽奖降低绑定 Debuff 概率。
3. 诅咒类奖励必须允许跳过。
4. Boss 奖励可强风险，但必须提供至少一个稳定选项。
5. 新手难度下，前 5 级不出现重度 Debuff。

---

## 7. 风险节点奖励规则

风险节点可以提高高品质奖励概率，但必须对应节点内压力。

| 风险等级 | 奖励提升 | Debuff 风险 |
|---|---|---|
| 低风险 | 紫 / 粉概率小幅提升 | 轻度 Debuff |
| 中风险 | 粉 / 红概率提升 | 中度 Debuff |
| 高风险 | 红 / 金概率提升 | 重度或诅咒 Debuff |

---

## 8. 角色联动

不同角色可以对风险奖励有特殊偏好：

| 角色方向 | 联动方式 |
|---|---|
| 新手均衡型 | 降低首次 Debuff 危险等级 |
| 状态连锁型 | 状态类 Debuff 可转化为状态反应加成 |
| 风险收益型 | Debuff 数量越多，输出越高 |
| 控制 / 生存型 | 可压制部分操作类和环境类 Debuff |

---

## 9. 禁止绑定规则

以下情况不建议绑定：

- 新手第一局前 3 次升级
- 教程节点
- 第一个武器奖励
- 玩家生命极低且没有恢复选项时
- 玩家已持有同类型重度 Debuff 时
- 当前 Debuff 会直接反制玩家唯一有效 build 时

---

## 10. 设计示例

### 示例 1：高攻击奖励

```text
奖励：破阵之力
品质：红
效果：物理伤害 +35%，击退 +20%
绑定 Debuff：防线空洞 II
代价：受到近战敌人伤害 +12%
适合：爆发流、物理近战
```

### 示例 2：状态奖励

```text
奖励：雷纹扩散
品质：粉
效果：感电链跳次数 +1
绑定 Debuff：电荷反噬 I
代价：未触发感电连锁时，下一次受到伤害 +5%
适合：连锁流
```

### 示例 3：诅咒奖励

```text
奖励：星裂契约
品质：金
效果：所有状态反应伤害 +50%
绑定 Debuff：星裂诅咒
代价：每次状态反应有 6% 概率对自身施加 1 层随机轻度状态
适合：高风险状态流
```
