# 抽奖 / 奖励系统设计 Prompts

以下 prompts 适合在 Codex 本地仓库环境中分阶段执行，用于生成或维护 `rewards/` 文件夹中的抽奖与奖励系统文档。

---

## Prompt 1：读取仓库并理解已有系统，不修改文件

```text
这是一个本地 Git 仓库中的游戏策划文档项目。请先读取当前仓库中的所有已有文件，尤其是：

1. 项目总策划案 / GDD
2. 元素系统 / 状态系统设计文档
3. weapons/ 武器设计文档，如果存在
4. characters/ 角色设计文档，如果存在
5. monsters/ 怪物设计文档，如果存在
6. nodes/ 节点 / 关卡路线系统文档，如果存在

本阶段只允许阅读和分析，不要创建、修改或删除任何文件。

请完成：
1. 总结当前项目的核心循环。
2. 总结当前局内成长来源。
3. 总结当前节点系统与奖励系统的关系。
4. 总结武器、状态、debuff、水晶、角色成长对抽奖系统的需求。
5. 分析适合本项目的 Roguelite / Action Survival / Brotato-like 奖励设计原则。
6. 判断当前仓库是否已经存在 `rewards/` 文件夹。
7. 输出后续设计抽奖 / 奖励系统必须遵守的设计约束。

注意：
- 不要修改文件。
- 不要创建文件。
- 不要提交 commit。
```

---

## Prompt 2：先提出系统框架，不写文件

```text
基于你刚才读取到的项目设定，请设计一套适用于本项目的“抽奖 / 奖励系统框架”。

本阶段不要创建、修改或删除任何文件，只输出框架供我确认。

框架需要覆盖：
1. 抽奖触发时机：升级、节点奖励、商店、事件、Boss、休息、精英。
2. 奖励类型：武器、武器升阶、状态核心、Buff、Debuff 修正、资源、水晶碎片、角色技能强化。
3. 奖励品质：蓝、紫、粉、红、金。
4. 抽奖池结构：通用池、武器池、状态池、角色倾向池、节点专属池、风险池、Boss 池。
5. 概率与权重：基础概率、角色倾向、水晶影响、当前 Build 影响、保底机制。
6. 风险收益：高品质奖励如何绑定 debuff 或风险条件。
7. 与节点系统的联动：不同节点给出不同奖励。
8. 与局外水晶系统的联动：水晶如何影响抽奖权重。
9. 与状态 / 武器 / 角色 / 怪物系统的联动。
10. 平衡性限制。

请参考 Roguelite、Action Survival、Brotato-like、Vampire Survivors-like 游戏的设计经验，但不要直接照搬具体系统。
```

## Prompt 3：正式创建 `rewards/` 文档

```text
我确认抽奖 / 奖励系统框架可以继续。

现在请在仓库根目录创建 `rewards/` 文件夹，并生成抽奖 / 奖励系统设计文档。

请至少创建以下文件：

1. `rewards/README.md`：系统总览
2. `rewards/lottery_reward_design_framework.md`：抽奖 / 奖励系统框架
3. `rewards/reward_pool_structure.md`：奖励池结构
4. `rewards/lottery_rules_and_probabilities.md`：抽奖规则与概率
5. `rewards/reward_quality_and_rarity.md`：奖励品质与稀有度
6. `rewards/risk_reward_and_debuff_binding.md`：风险收益与 Debuff 绑定
7. `rewards/shop_and_currency_rules.md`：商店与货币规则
8. `rewards/node_reward_integration.md`：节点奖励联动
9. `rewards/crystal_weight_rules.md`：水晶对抽奖权重的影响
10. `rewards/reward_item_examples.md`：奖励示例列表

要求：
- 保持中文策划文档风格。
- 与当前总策划、元素系统、武器、角色、怪物、节点系统保持一致。
- 不要删除或破坏已有文件。
- 机制要可开发落地。
- 每条规则都要说明设计目的。
- 高品质奖励不能无脑强，必须有风险、条件或机会成本。
```

## Prompt 4：自查与修复

```text
请检查 `rewards/` 文件夹中的所有 Markdown 文件。

重点检查：
1. 是否存在 README。
2. 是否覆盖抽奖触发、奖励池、品质概率、风险收益、商店货币、节点联动、水晶权重、奖励示例。
3. 是否与节点系统的节点类型一致。
4. 是否与元素 / 状态系统中的状态名称一致。
5. 是否与武器、角色、怪物设计没有冲突。
6. 是否存在过强奖励、无成本红金奖励、概率不合理、保底机制不清晰的问题。
7. 是否存在难以开发落地的复杂规则。

如果发现问题，请直接修复，并输出修复摘要。
```

## Prompt 5：最终摘要与 commit 建议

```text
请基于当前 Git 工作区，输出本次抽奖 / 奖励系统设计任务的最终修改摘要。

请列出：
1. 新增文件。
2. 修改文件。
3. `rewards/` 文件夹结构。
4. 系统核心规则摘要。
5. 与节点、武器、状态、角色、debuff、水晶系统的联动。
6. 建议的 Git commit message。

不要自动 commit，先让我确认。
```
