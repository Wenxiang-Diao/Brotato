# 怪物设计任务 Prompts

## Prompt 1：读取并理解已有设定，不修改文件
这是一个本地 Git 仓库中的游戏策划文档项目。请先读取当前仓库中的所有已有文件，尤其是项目总策划案、元素/状态系统设计、武器设计文档、characters/ 与 weapons/ 目录（如果存在）。本阶段只允许阅读和分析，不要创建、修改或删除任何文件。

请输出：当前项目核心玩法、状态/元素系统约束、战斗节奏约束、怪物设计应承担的功能、适合本项目的怪物设计原则，以及当前仓库是否已有 monsters/ 文件夹。

## Prompt 2：设计怪物设计框架，不创建文件
基于当前项目设定，请设计一个适用于《星链回响》的“怪物设计框架”。框架需要参考动作生存、Roguelite、Brotato-like、Vampire Survivors-like 游戏的敌人设计经验，但要适配本项目“元素反应构筑 + 风险管理 + 节点推进”的核心。

框架至少包含：基本信息、怪物概念、战斗定位、基础属性、移动/攻击模式、状态与元素关系、特殊机制、AI 行为、关卡投放、掉落与奖励、克制关系、平衡性备注、开发实现备注。

## Prompt 3：提出怪物名单，不创建文件
基于怪物设计框架和当前项目设定，请设计一组适用于 MVP 阶段的怪物名单。你需要自行决定数量，但要覆盖普通怪、精英怪、Boss、特殊机制怪。每个怪物先给出概要信息：名称、分类、定位、主要机制、状态倾向、推荐出现节点、主要威胁、克制方式。

要求：怪物之间机制差异明显；必须能测试玩家的走位、清怪、控制、状态联动、风险收益构筑；不要全是数值怪。

## Prompt 4：正式创建 monsters/ 文档
我确认怪物名单可以继续。请在仓库根目录创建 monsters/ 文件夹，创建 monsters/README.md、monster_design_framework.md，并为每个怪物单独创建 Markdown 文件。每个怪物文件必须按照已确认的框架展开。

要求：不要删除已有文件；保持中文策划文档风格；所有怪物要与元素/状态/武器/角色/build 系统联动；每个怪物都要有优势、弱点和平衡性限制；完成后输出新增文件列表和设计摘要。

## Prompt 5：自查和修复
请检查 monsters/ 目录所有 Markdown 文件，确认数量、命名、分类、字段、README 总览表一致。检查是否存在机制重复、过强、无弱点、无法开发落地、与状态系统冲突的问题。如发现问题请直接修复，并输出修复摘要。
