# ⚖️ 水晶系统平衡性与 UI/UX 规则

---

## 1. 平衡性目标

水晶系统要让玩家感到“越来越会构筑”，而不是“数值越来越碾压”。

衡量标准：

- 新玩家点了水晶后更容易理解构筑。
- 老玩家能通过水晶尝试不同路线。
- 高难度仍需要局内操作与决策。
- 单一分支不应成为绝对最优。

---

## 2. 数值控制原则

### 2.1 永久直接伤害限制

永久直接伤害提升应极少出现。

如果必须出现：

- 单节点不超过 3%~5%。
- 不应多个节点线性叠加成主要强度来源。
- 应放在较深层或附带条件。

### 2.2 权重提升限制

单个标签权重提升建议：

| 来源 | 建议上限 |
|---|---:|
| 单个节点 | 3%~5% |
| 单分支总和 | 10%~15% |
| 角色 + 水晶叠加 | 20% 以内 |

### 2.3 风险节点限制

风险收益节点必须有代价：

- 增加 Debuff 获得概率
- 降低商店治疗效率
- 提高修正成本
- 延迟奖励结算
- 提高敌人压力

---

## 3. 防止无聊的设计规则

避免节点大量写成：

```text
攻击力 +1%
生命值 +2%
暴击率 +1%
```

优先写成：

```text
感电链跳衰减降低
修正窗口出现替换选项概率提升
休息节点可能出现凝流专属修正
选择风险奖励后提高灼焰奖励权重
击败破甲敌人有概率返还金币
```

---

## 4. UI 设计

### 4.1 主界面结构

水晶界面建议分为：

- 中心星核
- 五个元素分支
- 当前资源栏
- 分支说明栏
- 推荐构筑标签
- 重置按钮

### 4.2 节点显示内容

每个节点点击后显示：

- 节点名称
- 当前等级 / 最大等级
- 效果说明
- 下一级效果
- 升级消耗
- 解锁条件
- 关联系统标签
- 平衡提示

### 4.3 新手引导

首次进入水晶界面时，只显示：

- 星核基础节点
- 一个推荐元素分支
- “推荐给新手”的提示

避免一次展示完整大树造成压力。

---

## 5. 反馈设计

升级节点时应有明显反馈：

- 水晶点亮
- 分支颜色变化
- 树根延展动画
- 简短音效
- 解锁节点时出现说明弹窗

分支颜色建议：

| 分支 | 颜色倾向 |
|---|---|
| 导能 | 蓝紫 / 电光 |
| 生息 | 绿色 / 花芽 |
| 凝流 | 冰蓝 / 水纹 |
| 灼焰 | 橙红 / 火星 |
| 结晶 | 金黄 / 晶石 |
| 星核 | 白金 / 星光 |

---

## 6. 信息透明度

水晶效果必须明确告诉玩家影响了什么。

推荐描述：

```text
“感电类奖励出现权重小幅提升”
“每章首次精英节点前，爆发类奖励更容易出现”
“修正窗口中更容易出现替换 Debuff 选项”
```

避免描述：

```text
“幸运提升”
“变得更强”
“元素之力增强”
```

---

## 7. 重置体验

重置功能应放在界面明显但不误触的位置。

确认弹窗应显示：

- 将返还哪些资源
- 返还比例
- 不返还哪些消耗
- 是否影响角色羁绊

默认建议不影响角色羁绊。

---

## 8. 数据验证指标

开发后可通过以下指标判断系统是否合理：

| 指标 | 目标 |
|---|---|
| 玩家第一局后是否能升级至少 1 个节点 | 是 |
| 玩家前 5 局是否能形成一个分支方向 | 是 |
| 单一分支选择率是否过高 | 不应超过 40% |
| 水晶升级后通关率是否暴涨 | 不应暴涨 |
| 玩家是否频繁重置 | 频繁说明树设计误导或成本不合理 |
| 高风险节点选择率 | 应有吸引力但不是必选 |
