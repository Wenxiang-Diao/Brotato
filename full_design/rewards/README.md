# 🎲 抽奖 / 奖励系统总览（Lottery & Reward System）

> 版本：v1.0  
> 优先级：P0  
> 适用阶段：MVP / Early Build  
> 依赖系统：战斗、节点、元素/状态、武器、角色、Debuff、水晶、商店、局外成长

---

## 1. 系统定位

抽奖 / 奖励系统是《星链回响》的局内构筑驱动器。它负责把战斗、节点路线、武器、状态、debuff、水晶和角色倾向串联成一次完整的 Roguelite 成长体验。

本系统的核心目标不是单纯给玩家更高数值，而是让玩家在每次奖励选择中形成明确判断：

> 我要更快成型，还是降低风险？  
> 我要强化已有流派，还是转向新的状态组合？  
> 我要拿高品质奖励并承受 debuff，还是选择稳定成长？

---

## 2. 核心原则

| 原则 | 说明 |
|---|---|
| 机制优先 | 奖励优先改变触发方式、状态交互、构筑方向，而不是只加攻击力 |
| 三选一决策 | 大多数奖励以 3 个选项呈现，制造清晰取舍 |
| Build 可读性 | 奖励需要显示标签，例如灼烧、感电、冻结、连锁、控制、风险等 |
| 风险收益 | 强奖励通常绑定 debuff、生命代价、资源消耗或触发条件 |
| 权重可控 | 角色、水晶、已有武器和状态会影响奖励池权重 |
| 防止卡死 | 通过保底、修正、重抽机制降低完全无法成型的概率 |
| 节点差异 | 不同节点提供不同奖励结构，路线选择必须有策略价值 |

## 4. MVP 抽奖触发点

| 触发点 | 主要奖励 | 设计目的 |
|---|---|---|
| 升级 | Buff / 状态核心 / 小型武器强化 | 高频构筑决策 |
| 普通战斗节点 | 基础奖励 + 小概率抽奖 | 稳定成长 |
| 精英节点 | 高品质奖励 + 风险选项 | 强化路线选择 |
| 抽奖节点 | 免费高价值三选一 | 构筑转折点 |
| 商店节点 | 购买、重抽、修正 | 玩家主动规划 |
| 事件节点 | 风险收益、特殊奖励 | 打破固定节奏 |
| 休息节点 | 恢复、移除 debuff、低价值奖励 | 风险修正 |
| Boss 节点 | 稀有奖励、水晶碎片、解锁 | 阶段性成长高潮 |

---

## 5. 初版奖励类型

| 类型 | 示例 | 核心作用 |
|---|---|---|
| 武器奖励 | 新武器、武器升阶材料 | 构筑基础 |
| 状态核心 | 灼烧核心、感电核心、冻结核心 | 强化状态体系 |
| Buff 模块 | 攻速、暴击、状态扩散、吸血 | 局内成长 |
| Debuff 修正 | 移除、替换、压缩 debuff | 风险管理 |
| 资源奖励 | 金币、经验、水晶碎片 | 经济与局外成长 |
| 角色强化 | 主动技能强化、终极技能充能 | 角色差异化 |
| 风险奖励 | 高品质强化 + 负面效果 | 高风险高收益 |

---

## 6. 推荐开发优先级

MVP 阶段优先实现：

1. 升级三选一奖励
2. 节点奖励表
3. 基础品质概率
4. 简单重抽机制
5. Debuff 绑定高品质奖励
6. 水晶权重加成的最小实现
7. 商店购买与刷新

后续版本再扩展：

- Build 智能推荐权重
- Boss 专属奖励池
- 角色专属奖励池
- 事件专属奖励链
- 图鉴发现型奖励
